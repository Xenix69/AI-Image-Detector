# 🔍 AI Image Detector

> A machine learning tool that detects whether an image is a real photograph or AI-generated — built as part of the Africa AI Hub AI Skills Immersion Programme (AISIP), Cohort 1.

---

## 📌 What It Does

Upload any image and the model will tell you:
- Whether it is **Real** or **AI-Generated**
- A **confidence score** showing how certain it is
- A **visual bar** representing the probability

Built to help combat AI-generated misinformation — a growing threat in Nigeria and across Africa, where deepfake images are increasingly used in political campaigns, fake news, and social media fraud.

---

## 🛠️ Built With

- **Python 3.13**
- **TensorFlow / Keras** — model training and inference
- **MobileNetV2** — transfer learning backbone
- **Gradio** — web interface
- **PIL / NumPy** — image preprocessing
- **CIFAKE Dataset** — 100,000 real and AI-generated images

## 🚀 Getting Started

### 1. Clone the repository
```bash
https://github.com/Xenix69/AI-Image-Detector.git
cd ai-image-detector
```

### 2. Install dependencies
```bash
pip install tensorflow pillow gradio numpy
```

### 3. Download the dataset
Get the CIFAKE dataset from Kaggle:
```
https://www.kaggle.com/datasets/birdy654/cifake-real-and-ai-generated-synthetic-images
```
Extract it and place the `train/` folder in the root of the project.

### 4. Run the notebook
Open `detector.ipynb` in Jupyter and run all cells top to bottom.

### 5. Launch the interface
The last cell launches the Gradio interface at:
```
http://127.0.0.1:7860
```
Open that link in your browser, upload an image and click **Detect**.

## 🧠 How It Works

### Model Architecture
```
Input Image (64x64x3)
        ↓
MobileNetV2 (pre-trained on ImageNet, frozen)
        ↓
GlobalAveragePooling2D
        ↓
Dense(64, ReLU)
        ↓
Dropout(0.3)
        ↓
Dense(1, Sigmoid)
        ↓
Output: probability score (0 = Real, 1 = AI-Generated)
```

### Training
| Setting | Value |
|---|---|
| Dataset | CIFAKE (100,000 images) |
| Train / Val Split | 80,000 / 20,000 |
| Image Size | 64 × 64 pixels |
| Batch Size | 32 |
| Epochs | 10 |
| Optimiser | Adam |
| Loss | Binary Cross-Entropy |
| Val Accuracy | ~85% |

### Why MobileNetV2?
The first model was a simple 3-layer CNN which overfitted — achieving 98% training accuracy and 0.05 loss but failing to generalise properly. Switching to MobileNetV2 transfer learning fixed this. A model pre-trained on millions of images already understands edges, textures, and shapes — it just needs to learn the specific difference between real and AI-generated content.

## ⚠️ Known Limitations

- Trained on CIFAKE (32×32 images upscaled to 64×64) — may struggle with high-resolution phone photos
- Training data was generated by Stable Diffusion only — images from Midjourney or DALL-E 3 may behave differently
- Runs locally only — no public URL yet
- Has not been tested on Nigerian news images or locally circulating deepfakes

---

## 🌍 African Relevance

This tool was built with the Nigerian context in mind:

- **Older Nigerians** are most vulnerable to deepfake misinformation — they rely on images and videos as proof and often lack the digital literacy to question what they see
- **Journalists and fact-checkers** can use this to verify images before publication
- **Runs on CPU** — no GPU or cloud infrastructure required, making it deployable on standard laptops across African institutions
